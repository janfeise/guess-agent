# Prompt 目录说明

这里存放猜词游戏后端使用的提示词文件。每个文件都对应一个独立职责，建议不要把所有规则混在一个 prompt 里，否则后续很难维护和调试。

## 文件说明

### `agent_decision.txt`

Agent 的决策提示词，用于让系统判断下一步应该做什么：

- 继续提问 `ask`
- 直接猜测 `guess`

它通常会结合当前历史、摘要和已有约束，让模型输出结构化结果，方便服务层决定后续流程。

### `game_rule.txt`

游戏规则提示词，用于描述猜词游戏的基础约束和通用规则。

这个文件适合放那些所有模型都必须遵守的规则，比如：

- 不能泄露答案
- 只能基于当前局信息推理
- 回答要稳定、简洁、克制

如果后面要统一全局规则，优先改这个文件。

### `game_start.txt`

开局生成提示词，用于在创建游戏时让小助手生成本局目标词。

它的目标是让模型输出一个符合约束的词语，通常只返回单个词，不要解释。

### `guess_system.txt`

主模型系统提示词，用于用户回合的问答和猜词推理。

这个文件通常负责约束主模型：

- 只能基于当前局历史回答问题
- 不能直接泄露答案
- 输出要尽量稳定
- 必要时按照结构化格式返回结果

如果你要控制用户提问阶段的回答逻辑，主要修改这个文件。

### `helper_system.txt`

小助手系统提示词，用于辅助类能力。

它通常承担以下职责：

- 开局取词
- 知识解释
- 辅助提示

这个文件应该和主模型的推理职责分开，避免小助手影响游戏公平性。

### `turn_router.txt`

用户输入意图识别提示词，用于判断用户当前输入属于哪一种类型：

- `question`：提问
- `guess`：猜测
- `invalid`：无效输入

这个文件是猜词核心流程里很重要的一环，作用是把自由文本先分类，再交给后续流程处理。

## 推荐使用顺序

一个比较清晰的调用顺序是：

1. 游戏开始时，读取 `game_start.txt`
2. 用户输入进入系统时，先通过 `turn_router.txt` 做意图识别
3. 如果是正常提问，使用 `guess_system.txt` 生成回答
4. 如果需要 Agent 自主决策，读取 `agent_decision.txt`
5. 如果是辅助查询或开局相关能力，使用 `helper_system.txt`

`game_rule.txt` 可以作为公共规则，在多个 prompt 里复用或拼接。

## 维护建议

- 每个文件只负责一个职责，不要跨职责混写
- 修改规则时优先改对应文件，不要直接把规则硬编码进 Python 代码
- 如果某个 prompt 的职责不清晰，先在这里补充说明，再改代码
- 新增 prompt 文件时，也要同步更新本 README
