## 概述

本文档描述了"学科猜词大挑战"（guess-agent）项目的后端 REST API 接口规范。该系统实现了一个以 AI Agent 为核心的对抗式猜词游戏系统，支持用户 vs AI 的多轮问答互动。

### 核心特性

- 🎮 **对抗式游戏玩法**: 用户与 AI 轮流提问和猜测
- 🔐 **加密机制**: 游戏答案全程加密存储，确保公平性
- 🎓 **学科词库**: 支持计算机、数学、历史等多个学科领域
- 🤖 **AI 辅助**: 集成 AI Agent 进行智能推理和决策
- 📊 **游戏记录**: 完整的回合历史和推理过程记录

### 技术栈

- **Framework**: FastAPI (Python 3.8+)
- **Database**: MongoDB
- **LLM**: DeepSeek API、OpenAI-compatible
- **Encryption**: Fernet (cryptography library)

### 2. 创建游戏

#### POST `/api/v1/games`

**描述**: 创建一个新的游戏实例，初始化游戏状态，系统自动生成 AI 答案

**请求体**:

```json
{
  "owner_id": "user123",
  "user_word": "算法",
  "difficulty": "medium"
}
```

**请求参数说明**:

| 参数         | 类型   | 是否必需 | 说明                | 约束                                             |
| ------------ | ------ | -------- | ------------------- | ------------------------------------------------ |
| `owner_id`   | string | 否       | 游戏创建者的用户 ID | 可以为空，用于区分不同玩家                       |
| `user_word`  | string | 是       | 玩家输入的答案词    | 长度 1-10 字符，不能为空                         |
| `difficulty` | string | 否       | 游戏难度            | 可选值: `easy`, `medium`, `hard`，默认: `medium` |

**响应 (201 Created)**:

```json
{
  "game_id": "507f1f77bcf86cd799439011",
  "status": "active",
  "round_count": 1,
  "user_word": "算法",
  "difficulty": "medium",
  "system_word_encrypted": "gAAAAABmyZ5L...",
  "created_at": "2026-04-13T12:34:56.789123+00:00"
}
```

**响应字段说明**:

| 字段                    | 类型   | 说明                                 |
| ----------------------- | ------ | ------------------------------------ |
| `game_id`               | string | 游戏的唯一标识符（MongoDB ObjectId） |
| `status`                | string | 游戏状态，当前固定为 `active`        |
| `round_count`           | int    | 当前轮次，初始值为 1                 |
| `user_word`             | string | 玩家输入的词（反显，确认无误）       |
| `difficulty`            | string | 应用的难度等级                       |
| `system_word_encrypted` | string | AI 答案的密文，前端无需关注此字段    |
| `created_at`            | string | 游戏创建时间（ISO 8601 格式）        |

**错误响应**:

- `400 Bad Request` - 参数验证失败
  ```json
  {
    "detail": "user_word should not be empty"
  }
  ```
- `500 Internal Server Error` - 服务器错误
  ```json
  {
    "detail": "Failed to create game"
  }
  ```

**业务规则**:

1. **答案长度限制**: 用户输入的答案长度必须在 1-10 个字符内
2. **难度等级**:
   - `easy`: 针对初学者，AI 可能会给出更多提示
   - `medium`: 标准难度（默认）
   - `hard`: 高难度，AI 会尽量隐藏答案特征
3. **自动生成 AI 答案**: 系统会根据用户输入的答案所属学科自动为 AI 生成一个答案词
4. **加密**: 生成的系统答案立即加密存储，在游戏结束前不可解密

**使用场景**:

1. 用户在前端点击"开始游戏"后，输入自己要给 AI 猜的词
2. 前端调用此接口创建游戏
3. 获取 `game_id` 后，前端切换到游戏界面准备开始问答

### 3. 提交游戏回合

#### POST `/api/v1/games/{game_id}/turns`

**描述**: 提交玩家在当前回合的输入（提问、回答或判断），系统处理并返回 AI 的响应

**URL 参数**:

| 参数      | 类型   | 说明                          |
| --------- | ------ | ----------------------------- |
| `game_id` | string | 游戏 ID（由创建游戏接口返回） |

**请求体**:

```json
{
  "turn_type": "input",
  "content": "这个东西是有形的吗？",
  "mode": "agent"
}
```

**请求参数说明**:

| 参数        | 类型    | 是否必需 | 说明                          | 可选值                     |
| ----------- | ------- | -------- | ----------------------------- | -------------------------- |
| `turn_type` | string  | 是       | 回合类型                      | `input`, `answer`, `judge` |
| `content`   | string  | 否\*     | 玩家输入内容（提问或答案）    | -                          |
| `question`  | string  | 否       | 兼容旧客户端的输入字段        | -                          |
| `judgement` | boolean | 否\*     | 是否确认/同意系统的猜测或回答 | `true` / `false`           |
| `mode`      | string  | 否       | 回答模式                      | `agent`（默认）, `helper`  |

**`turn_type` 详解**:

| 值       | 触发条件       | 字段说明    | 说明                                                |
| -------- | -------------- | ----------- | --------------------------------------------------- |
| `input`  | 用户提问或猜测 | `content`   | 玩家输入是/否问题或目标词的猜测                     |
| `answer` | 系统提出问题后 | `content`   | 玩家回答系统提出的是/否问题                         |
| `judge`  | 系统做出猜测后 | `judgement` | 玩家确认系统的猜测是否正确（true 正确，false 错误） |

**`mode` 说明**:

- `agent`: 使用主 AI Agent 处理请求（默认，用于游戏进行时）
- `helper`: 使用辅助模型处理请求（仅用于知识查询，不参与游戏推理）

**响应 (200 OK)**:

```json
{
  "game_id": "507f1f77bcf86cd799439011",
  "status": "active",
  "phase": "waiting_answer",
  "result": "ask",
  "answer": null,
  "response_text": "这个东西是有形的吗？让我想想...是的，这个东西确实是有形的。",
  "hint": "具有物理形态",
  "confidence": 0.85,
  "history": [
    {
      "round_no": 1,
      "actor": "user",
      "turn_type": "question",
      "input_text": "这个东西是有形的吗？",
      "answer_text": "是的",
      "answer_label": "yes",
      "confidence": 0.9,
      "source_word_owner": "user",
      "visible_to_agent": true,
      "visible_to_player": true
    },
    {
      "round_no": 1,
      "actor": "system",
      "turn_type": "question",
      "input_text": "你认为它与计算机相关吗？",
      "source_word_owner": "system",
      "visible_to_agent": true,
      "visible_to_player": true
    }
  ]
}
```

**响应字段说明**:

| 字段            | 类型           | 说明                                                         |
| --------------- | -------------- | ------------------------------------------------------------ |
| `game_id`       | string         | 游戏 ID                                                      |
| `status`        | string         | 游戏状态（`active`, `finished_win`, `finished_loss` 等）     |
| `phase`         | string         | 当前游戏阶段                                                 |
| `result`        | string         | 当前处理结果（`ask`, `guess`, `answer` 等）                  |
| `answer`        | string \| null | LLM 对用户意图的理解结果（`yes`, `no`, `unknown` 或 `null`） |
| `response_text` | string \| null | 返回给用户的自然语言回复                                     |
| `answer_text`   | string \| null | 兼容旧客户端的字段                                           |
| `hint`          | string \| null | 简短提示（可选）                                             |
| `confidence`    | float          | 系统对回答的置信度（0.0 - 1.0）                              |
| `history`       | array          | 游戏历史记录数组                                             |

**`phase` 可能的值**:

| 阶段                 | 说明                            |
| -------------------- | ------------------------------- |
| `user_turn`          | 用户回合（等待用户提问或猜测）  |
| `waiting_answer`     | 等待用户回答系统的提问          |
| `awaiting_judgement` | 等待用户判断系统的猜测是否正确  |
| `finished_win`       | 游戏结束 - 系统猜测正确         |
| `finished_loss`      | 游戏结束 - 轮次用尽，系统未猜对 |

**历史记录项字段说明**:

| 字段                | 类型    | 说明                                                      |
| ------------------- | ------- | --------------------------------------------------------- |
| `round_no`          | int     | 轮次号                                                    |
| `actor`             | string  | 执行者（`user` 或 `system`）                              |
| `turn_type`         | string  | 回合类型（`question`, `answer`, `ask`, `guess`, `judge`） |
| `input_text`        | string  | 输入内容                                                  |
| `answer_text`       | string  | 回答内容                                                  |
| `answer_label`      | string  | 回答标签（`yes`, `no`, `unknown`）                        |
| `confidence`        | float   | 置信度                                                    |
| `source_word_owner` | string  | 答案来源（`user` 或 `system`）                            |
| `visible_to_agent`  | boolean | 是否对 AI 可见                                            |
| `visible_to_player` | boolean | 是否对玩家可见                                            |

**错误响应**:

- `400 Bad Request` - 业务逻辑错误
  ```json
  {
    "detail": "Game not found: invalid_game_id"
  }
  ```
- `500 Internal Server Error` - 服务器错误
  ```json
  {
    "detail": "Failed to submit turn"
  }
  ```

**游戏流程与轮次状态机**:

```
START (创建游戏)
  ↓
user_turn (用户提问或猜测)
  ├─ 提问 → waiting_answer (系统等待用户回答)
  │         ↓
  │    waiting_answer (用户回答) → user_turn (继续)
  │
  └─ 猜测 → awaiting_judgement (系统等待判断)
           ↓
      awaiting_judgement (用户判断)
           ├─ 正确 → finished_win (游戏赢)
           └─ 错误 / 其他 → user_turn (继续)
```

**轮次锁机制说明**:

系统采用严格的轮次锁机制确保游戏流程的完整性：

- `metadata.phase`: 当前游戏阶段
- `metadata.next_actor`: 下一个行动者（`user` 或 `system`）
- `metadata.expected_turn_type`: 期望的回合类型
- `metadata.last_actor`: 上一个行动者

系统会在每个请求中校验当前 phase 和 turn_type 是否匹配，如果不匹配会返回拒绝。

**使用场景**:

1. **用户提问**: `turn_type="input"`, `content="问题内容"`
2. **用户回答系统提问**: `turn_type="answer"`, `content="是/不是"`
3. **用户猜测答案**: `turn_type="input"`, `content="猜测的词"`
4. **用户确认系统猜测**: `turn_type="judge"`, `judgement=true/false`

## 游戏流程说明

### 完整游戏流程

```
1. 前端用户输入 → 调用 POST /api/v1/games (创建游戏)
   └─ 返回: game_id, 系统词密文

2. 前端进入游戏界面 → 阶段: user_turn
   └─ 等待用户提问 (turn_type="input")

3. 用户提问 → 调用 POST /api/v1/games/{game_id}/turns
   ├─ 请求: { "turn_type": "input", "content": "是..."问题 }
   ├─ 处理: LLM 分析问题，生成是/否答案
   └─ 返回: 阶段转入 waiting_answer, 返回系统回答

4. 用户回答系统问题 (如果 pending_question 不为空)
   ├─ 请求: { "turn_type": "answer", "content": "是/不是" }
   ├─ 处理: 解析用户意图，更新 reasoning_history
   └─ 返回: 阶段转入 user_turn，继续

5. 用户尝试猜测答案 (turn_type="input" with 目标词)
   ├─ 请求: { "turn_type": "input", "content": "猜测的词" }
   ├─ 判定: 后端比较解密后的答案
   ├─ 错误: → 阶段转入 awaiting_judgement, LLM 反过来猜测
   └─ 正确: → 游戏结束, status="finished_win"

6. 用户判断系统猜测结果
   ├─ 请求: { "turn_type": "judge", "judgement": true/false }
   ├─ true: 游戏结束, status="finished_win"
   └─ false: 阶段转入 user_turn，继续

7. 游戏结束 (reach max round or 猜对)
   └─ 前端展示结果、历史记录、分析
```

### 规则要点

1. **严格轮次制**: 每次只能提交一种类型的信息
2. **自动轮流**: 系统自动在用户和 AI 间切换
3. **答案加密**: 所有答案全程加密，游戏过程中不可见
4. **提示隐藏**: 系统提示在返回给前端前会清理敏感字段（result）
5. **最多轮次**: 默认 8 轮，超过后游戏失败

## 示例

### Python 示例

```python
import requests
import json

BASE_URL = "http://localhost:8000/api/v1"

# 1. 创建游戏
create_game_resp = requests.post(
    f"{BASE_URL}/games",
    json={
        "owner_id": "user123",
        "user_word": "算法",
        "difficulty": "medium"
    }
)
game_data = create_game_resp.json()
game_id = game_data["game_id"]
print(f"✓ 游戏创建成功: {game_id}")

# 2. 提问
submit_turn_resp = requests.post(
    f"{BASE_URL}/games/{game_id}/turns",
    json={
        "turn_type": "input",
        "content": "这个东西是有形的吗？",
        "mode": "agent"
    }
)
result = submit_turn_resp.json()
print(f"系统回答: {result['response_text']}")
print(f"当前阶段: {result['phase']}")

# 3. 回答系统提问
answer_resp = requests.post(
    f"{BASE_URL}/games/{game_id}/turns",
    json={
        "turn_type": "answer",
        "content": "是",
        "mode": "agent"
    }
)
result = answer_resp.json()
print(f"阶段转换: {result['phase']}")

# 4. 健康检查
health_resp = requests.get(f"{BASE_URL}/health")
health_data = health_resp.json()
print(f"服务状态: {health_data['status']}")
print(f"数据库状态: {health_data['database']}")
```

### JavaScript/TypeScript 示例

```typescript
const BASE_URL = "http://localhost:8000/api/v1";

// 1. 创建游戏
async function createGame(userWord: string, difficulty: string = "medium") {
  const response = await fetch(`${BASE_URL}/games`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      owner_id: "user123",
      user_word: userWord,
      difficulty: difficulty,
    }),
  });
  const game = await response.json();
  return game.game_id;
}

// 2. 提交回合
async function submitTurn(
  gameId: string,
  turnType: "input" | "answer" | "judge",
  content?: string,
  judgement?: boolean,
) {
  const body: any = {
    turn_type: turnType,
    mode: "agent",
  };

  if (turnType === "input" || turnType === "answer") {
    body.content = content;
  } else if (turnType === "judge") {
    body.judgement = judgement;
  }

  const response = await fetch(`${BASE_URL}/games/${gameId}/turns`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
    },
    body: JSON.stringify(body),
  });
  return await response.json();
}

// 3. 健康检查
async function checkHealth() {
  const response = await fetch(`${BASE_URL}/health`);
  return await response.json();
}

// 使用示例
(async () => {
  // 检查服务
  const health = await checkHealth();
  console.log("✓ 服务状态:", health.status);

  // 创建游戏
  const gameId = await createGame("算法", "medium");
  console.log("✓ 游戏创建:", gameId);

  // 用户提问
  const turn1 = await submitTurn(gameId, "input", "这个东西是有形的吗？");
  console.log("系统回答:", turn1.response_text);
  console.log("当前阶段:", turn1.phase);

  // 用户回答
  const turn2 = await submitTurn(gameId, "answer", "是");
  console.log("阶段转换:", turn2.phase);
})();
```

## 常见问题

### Q1: 系统如何保证答案安全？

**A**: 所有答案（用户词和系统词）使用 Fernet 加密存储。密钥通过 SHA256 哈希派生自密码。游戏过程中服务端通过解密方式比对答案，而不是直接暴露。

### Q2: 前端如何获取最终答案？

**A**: 游戏结束时，前端需要向用户询问用户答案的密码，然后在前端本地解密 `user_word_encrypted` 进行验证。系统词由后端在游戏结束时根据游戏结果返回（需实现此接口）。

### Q3: 回合数上限是多少？

**A**: 默认为 8 轮。超过此上限后游戏自动失败。此参数配置可在 `memory_policy` 中调整。

### Q4: 系统提示会暴露答案吗？

**A**: 系统会自动清理返回给前端的历史记录中的 `result` 字段，避免泄露 LLM 的推理过程中可能包含的答案信息。

### Q5: 何时触发系统反向猜测？

**A**: 当用户提交猜测，且猜测答案与系统词不符时，系统会根据前面的问答历史反向推理用户答案的可能性，并提出自己的猜测。

### Q6: "小助手"模式与普通模式有何区别？

**A**:

- `mode="agent"`: 使用主 AI Agent，参与游戏推理
- `mode="helper"`: 使用轻量化助手模型，仅用于知识查询和学习支持，不参与游戏判定

### Q7: 如何处理网络超时？

**A**: 建议设置合理的请求超时时间（建议 30-60 秒）。后端 LLM 调用超时默认为 180 秒。若频繁超时，检查网络连接和 API 配额。
