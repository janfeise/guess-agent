## 🔀 数据流向

### 游戏初始化流程

```
┌─────────┐
│  用户   │
└────┬────┘
     │ 1. 输入单词 + 选择难度
     ↓
┌──────────────────┐
│   index.vue      │
│ (首页)           │
│                  │
│ form: {          │
│   startWord,     │
│   difficulty     │
│ }                │
└────┬─────────────┘
     │ 2. 点击"开始游戏"
     │
     ├─→ validateInput()
     │
     ├─→ createNewGame()  ← gameLogic.js
     │   {
     │     id, startWord, difficulty,
     │     questionsUsed, guessLimit, ...
     │   }
     │
     ├─→ uni.navigateTo('/pages/gamePlay/gamePlay')
     │
     ↓
┌──────────────────┐
│  gamePlay.vue    │
│ (游戏进行)       │
│                  │
│ currentGame: {}  ← 接收初始化数据
│ messages: []     │
│ userInput: ""    │
└──────────────────┘
```

### 聊天交互流程

```
用户输入消息
    ↓
detectInputType()  ← 判断是"问题"还是"猜测"
    ↓
检查限制:
├─ 问题 → questionsUsed < questionLimit?
└─ 猜测 → guessesUsed < guessLimit?
    ↓
    ├─ YES ↓
    │   ├─ 计数器 ++
    │   ├─ generateAIResponse()
    │   ├─ delay (800-2000ms)
    │   ├─ messages.push(AIMsg)
    │   ├─ calculateAIConfidence()
    │   ├─ showHunchCard() (每3次提问)
    │   └─ saveCurrentGame()
    │
    └─ NO ↓
        └─ uni.showToast('名额已用完')

判断游戏是否结束:
├─ 猜测正确 → 胜利 → result页
├─ 猜测错误 & 还有机会 → 提示"再试试"
└─ 猜测错误 & 名额用完 → 失败 → result页
```

### 结果统计流程

```
游戏结束
    ↓
createGameResult()
    {
      gameId, startWord, difficulty,
      finalAnswer, isWon,
      roundsUsed, questionsUsed, guessesUsed,
      totalTime, aiConfidence, timestamp
    }
    ↓
saveGameResult()
    ├─ 保存到 gameHistory
    ├─ 更新 userStats
    │   ├─ totalGames++
    │   ├─ 根据isWon更新胜负统计
    │   ├─ 按难度分别统计
    │   ├─ 更新最佳成绩 (bestTime, minRounds)
    │   └─ 记录最后游戏时间
    └─ localStorage.setItem()
        ↓
    显示 result 页面
```

## 🏗️ 核心模块设计

### 1️⃣ gameLogic.js (游戏逻辑引擎)

**职责**：

- 游戏对象创建和管理
- 难度配置和参数供应
- AI响应生成（模拟）
- 输入类型识别
- 游戏结果计算

**关键函数**：

| 函数                      | 输入                  | 输出                  | 作用             |
| ------------------------- | --------------------- | --------------------- | ---------------- |
| `createNewGame()`         | startWord, difficulty | game object           | 初始化新游戏     |
| `detectInputType()`       | userInput             | 'question' \| 'guess' | 判断提问还是猜测 |
| `generateAIResponse()`    | roundNum, difficulty  | response object       | 生成AI回答       |
| `generateHunchHint()`     | word                  | hint string           | 生成直觉提示     |
| `calculateAIConfidence()` | 游戏参数              | 0-100                 | 计算AI信心指数   |
| `createGameResult()`      | game, answer, isWon   | result object         | 生成游戏成绩     |

**状态管理**：

```javascript
深度对象: game = {
  id: String, // 唯一标识
  startWord: String, // 用户输入的起始词
  difficulty: String, // easy/medium/hard
  questionsUsed: Number, // 已用提问数
  questionLimit: Number, // 提问限制
  guessesUsed: Number, // 已用猜测数
  guessLimit: Number, // 猜测限制
  roundNum: Number, // 轮数计数
  startTime: Number, // 游戏开始时间戳
  messages: Array, // 聊天消息
  status: String, // playing/won/lost
  result: Object | null, // 游戏结果
};
```

---

### 2️⃣ storage.js (数据持久化层)

**职责**：

- 游戏历史管理
- 用户统计更新
- 当前游戏保存（断点恢复）
- 本地缓存操作

**存储结构**：

```
localStorage {
  'hunch_game_history': [         // 最多50条
    {
      gameId, startWord, difficulty,
      finalAnswer, isWon, roundsUsed,
      questionsUsed, guessesUsed,
      totalTime, aiConfidence, timestamp
    },
    ...
  ],
  'hunch_user_stats': {
    totalGames: Number,
    winsCount: Number,
    lossesCount: Number,
    byDifficulty: {
      easy: { played, won },
      medium: { played, won },
      hard: { played, won }
    },
    bestTime: Number,     // 最短用时(ms)
    minRounds: Number,    // 最少轮数
    lastPlayTime: Number  // 最后游戏时间
  },
  'hunch_current_game': {           // 临时
    // 当前游戏对象（断点恢复用）
  }
}
```

**关键API**：

| 函数                     | 作用                    |
| ------------------------ | ----------------------- |
| `saveGameResult(result)` | 保存游戏结果 → 更新统计 |
| `getGameHistory(filter)` | 获取历史 (all/won/lost) |
| `getUserStats()`         | 获取用户统计            |
| `getWinRate()`           | 计算胜率百分比          |
| `saveCurrentGame(game)`  | 保存当前游戏（临时）    |
| `clearGameHistory()`     | 清除所有历史记录        |

## 📈 性能考虑

### 优化点

1. **消息列表** (gamePlay)
   - 限制消息数量 ≤ 100条
   - 超出时自动清空较早消息
   - 考虑虚拟滚动优化

2. **历史记录** (history)
   - 只保存最近50条
   - 使用筛选减少DOM节点
   - 分页加载（未来）

3. **图片加载**
   - AI头像使用CDN缓存
   - 考虑本地缓存头像

4. **状态管理**
   - 游戏进行中只在内存保存
   - 结束后主动清空currentGame
