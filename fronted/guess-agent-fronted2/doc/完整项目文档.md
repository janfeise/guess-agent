## 项目概述

**HUNCH** 是一个创新的 AI 辅助的单词游戏应用，基于 Google Stitch 原型全面重构为完整的 uni-app 项目。

### 核心特性

- ✅ **AI 交互系统**：与 AI 助手进行实时问答
- ✅ **多难度等级**：简单/普通/困难三个难度
- ✅ **游戏历史记录**：本地存储所有游戏数据
- ✅ **用户统计**：胜率、最佳成绩、难度分析
- ✅ **完整交互反馈**：Loading、Toast、动画过渡
- ✅ **响应式设计**：完美适配各种移动设备

## 页面说明

### 1️⃣ 首页 (pages/index/index.vue)

**功能**：游戏初始化界面，玩家设置一个初始单词和难度等级。

**核心交互**：

- `文本输入` → 输入起始单词（如"大自然"、"自行车"）
- `难度选择` → 单选按钮切换难度（简单/普通/困难）
- `开始游戏` → 表单验证 + Toast提示 + 导航至gamePlay
- `最近游戏` → 点击快速查看历史记录
- `玩法说明` → 跳转至规则页面

**技术实现**：

```javascript
// 游戏初始化流程
createNewGame() → uni.showToast() → uni.navigateTo() → 传递game数据到gamePlay页
```

**难度等级配置**：

- **简单**: 15问题名额, 5次猜测
- **普通**: 10问题名额, 3次猜测
- **困难**: 7问题名额, 2次猜测

---

### 2️⃣ 游戏进行页 (pages/gamePlay/gamePlay.vue)

**功能**：实时对话交互，用户与AI进行问答和猜测。

**核心交互**：

| 交互            | 说明                               |
| --------------- | ---------------------------------- |
| **提问**        | 用户提出概念性问题，AI给出精简回答 |
| **猜测**        | 输入目标词语，判断是否正确         |
| **Hunch提示**   | 每3次提问后，AI给出隐喻性直觉提示  |
| **自动滚动**    | 新消息自动滚动到底部               |
| **Loading反馈** | 等待AI响应时显示加载指示器         |

**消息流程**：

```
用户输入 → 类型判断 (问题/猜测) → 限制检查 → AI生成响应 → 延迟 → 显示消息 → 状态更新
```

**类型判断逻辑**：

- 以"吗、呢、？"结尾 → **提问**
- 包含"是否、能否、会不会" → **提问**
- 其他输入 → **猜测**

**游戏状态管理**：

- 轮次自动计数 (round++)
- 提问/猜测次数递增
- AI信心指数动态计算
- 消息历史实时保存

---

### 3️⃣ 结果页 (pages/result/result.vue)

**功能**：展示游戏成果，包括答案、统计数据、AI信心指数。

**界面组成**：
| 区域 | 说明 |
|------|------|
| **胜负横幅** | 大标题 + 描述文案（胜/负） |
| **正确答案卡** | 显示目标词 + 相关emoji |
| **统计结果** | 轮次/时间/信心指数/提问数 |
| **行动按钮** | "再玩一次" / "返回首页" |
| **分享统计** | 可复制的成绩文案 |

**数据展示**：

```
结果类型        |  表示方式
--------------|-----------
胜利事实        |  banner绿色 + 胜利emoji
失败事实        |  banner橙色 + 失败emoji
总用时          |  格式: m:ss
AI信心指数      |  0-100% 百分比条
轮数统计        |  数字展示
```

---

### 4️⃣ 历史记录页 (pages/history/history.vue)

**功能**：管理、查看和筛选所有历史游戏记录。

**核心交互**：

| 交互         | 功能                             |
| ------------ | -------------------------------- |
| **筛选标签** | 全部 / 获胜 / 落败               |
| **历史卡片** | 点击查看游戏详情                 |
| **统计面板** | 展示总局数、胜场、胜率、最少轮数 |
| **空状态**   | 无记录时提示"开始游戏"按钮       |

**数据过滤**：

```javascript
// 根据选中的filter值实时过滤游戏列表
all → 显示所有游戏
won → 显示isWon=true的游戏
lost → 显示isWon=false的游戏
```

---

### 5️⃣ 规则页 (pages/rules/rules.vue)

**功能**：详细说明游戏规则、玩法技巧和直觉提示机制。

**内容组织**：
| 区域 | 内容 |
|------|------|
| **三阶段说明** | 提问 → 推测 → 确认 |
| **核心规则** | 4条详细规则 |
| **直觉解释** | 什么是Hunch提示 + 示例 |
| **玩家贴士** | 4条实战建议 |
| **行动按钮** | 开始挑战 |

## 交互逻辑

### ⚡ 游戏流程图

```
┌─────────────┐
│    首页      │ ← 输入单词+选择难度
└──────┬──────┘
       │ 点击"开始游戏"
       ↓
┌─────────────────────────────────────┐
│   创建游戏对象 (createNewGame)       │
│   - 初始化难度参数                  │
│   - 生成游戏ID                      │
│   - 记录开始时间                    │
└──────┬──────────────────────────────┘
       │ 保存游戏状态、导航
       ↓
┌──────────────────────────────────────┐
│     gamePlay 页面 (游戏进行)          │
│   ┌────────────────────────────────┐ │
│   │ 用户输入 (提问/猜测)           │ │
│   └────────┬─────────────────────┘ │
│            ↓                         │
│   输入类型判断 (detectInputType)     │
│   ├─ 问题 → questionsUsed++         │
│   └─ 猜测 → guessesUsed++           │
│            ↓                         │
│   检查限制 & 生成AI响应             │
│   ├─ generateAIResponse()           │
│   ├─ 延迟 800-2000ms               │
│   └─ 添加消息到列表                 │
│            ↓                         │
│   状态更新                          │
│   ├─ 轮次 ++                        │
│   ├─ 保存当前游戏状态               │
│   ├─ 显示Hunch (每3次)             │
│   └─ 自动滚动到底部                 │
│            ↓                         │
│   循环直到游戏结束或违反限制        │
└──────┬───────────────────────────────┘
       │
       ├─ 🎉 用户猜对 → createGameResult(won=true)
       │
       ├─ 😞 用户猜数用完 → createGameResult(won=false)
       │
       └─ saveGameResult() → 本地存储
              ↓
        ┌─────────────┐
        │  结果页     │  ← 展示成绩、统计
        └─────────────┘
```

### 🔄 状态转移图

```
游戏状态 (GAME_STATUS)
├─ PLAYING    → 游戏进行中
├─ WON        → 玩家胜利
└─ LOST       → 玩家失败

游戏阶段 (GAME_PHASES)
├─ THINKING       → AI思考中
├─ USER_TURN      → 等待用户输入
├─ AI_RESPONSE    → AI正在回复
└─ RESULT         → 游戏已结束
```

```javascript

//   questionsUsed: 0,
//   questionLimit: 10,
//   guessesUsed: 0,
//   guessLimit: 3,
//   roundNum: 0,
//   status: 'playing',
//   messages: [],
//   ...
// }
```

#### 判断输入类型

```javascript
detectInputType("它是生物吗?"); // => 'question'
detectInputType("是钥匙"); // => 'guess'
```

#### 生成AI响应

```javascript
generateAIResponse(3, "medium");
// => { text: "你在正确的方向上。", delay: 1243 }
```

#### 生成直觉提示

```javascript
generateHunchHint("自行车");
// => "多考虑那些依靠人力而非热量的工具。"
```

#### 计算AI信心指数

```javascript
calculateAIConfidence(5, 8, 1, "medium");
// => 68 (百分比)
```

#### 创建游戏结果

```javascript
const result = createGameResult(game, "BICYCLE", true);

//   questionsUsed: 8,
//   guessesUsed: 1,

```
