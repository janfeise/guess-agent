## 三大核心接口

### 1️⃣ 健康检查 (无需授权)
```
GET /health
```
**用途**: 检查服务和数据库连接状态  
**返回**: 状态、时间戳

---

### 2️⃣ 创建游戏
```
POST /games
```

**发送**:
```json
{
  "owner_id": "user123",           // 可选
  "user_word": "算法",             // 必需，1-10 字符
  "difficulty": "medium"           // 可选: easy/medium/hard
}
```

**获得**:
```json
{
  "game_id": "507f1f77bcf86cd799439011",
  "status": "active",
  "user_word": "算法",
  "system_word_encrypted": "gAAAAABmyZ5L...",
  "created_at": "2026-04-13T12:34:56Z"
}
```

**错误**: 
- `400` - 答案为空或超长
- `500` - 服务器错误

---

### 3️⃣ 提交游戏回合 (核心接口)
```
POST /games/{game_id}/turns
```

#### 场景 A: 用户提问
```json
{
  "turn_type": "input",
  "content": "这个东西是有形的吗？",
  "mode": "agent"
}
```
✅ **转换阶段**: `user_turn` → `awaiting_answer`

#### 场景 B: 用户回答系统问题
```json
{
  "turn_type": "answer",
  "content": "是",
  "mode": "agent"
}
```
✅ **转换阶段**: `awaiting_answer` → `user_turn`

#### 场景 C: 用户猜测
```json
{
  "turn_type": "input",
  "content": "算法",
  "mode": "agent"
}
```
✅ **结果**: 
- 正确 → `finished_win` 
- 错误 → `awaiting_judgement` (系统反向猜测)

#### 场景 D: 用户判断系统猜测
```json
{
  "turn_type": "judge",
  "judgement": true,
  "mode": "agent"
}
```
✅ **结果**:
- `true` → `finished_win`
- `false` → `user_turn` (继续游戏)

**统一响应结构**:
```json
{
  "game_id": "...",
  "status": "active",                    // active/finished_win/finished_loss
  "phase": "awaiting_answer",            // 当前阶段
  "result": "ask",                       // 处理结果
  "response_text": "是的，...",          // 返回给用户的文本
  "hint": "具有物理形态",                // 可选提示
  "confidence": 0.85,                    // 置信度 0-1
  "history": [...]                       // 完整游戏历史
}
```

---

## 游戏阶段状态机

```
START (创建游戏)
   ↓
┌─────────────────────┐
│  user_turn          │ ← 用户提问或猜测
└─────────────────────┘
   ├─ 提问 → awaiting_answer
   │         └─ 用户回答 → user_turn
   └─ 猜测 → awaiting_judgement
             └─ 用户判断 → finished_win 或 user_turn

最多 8 轮 → finished_loss (轮次用尽)
```

## 数据类型速查

### `turn_type` 值
| 值 | 何时使用 | 配套字段 |
|----|---------|---------|
| `input` | 提问或猜测 | `content` |
| `answer` | 回答系统问题 | `content` |
| `judge` | 判断系统猜测 | `judgement` |

### `mode` 值
| 值 | 说明 |
|----|------|
| `agent` | 使用主 AI（游戏模式） |
| `helper` | 使用辅助模型（学习辅助） |

### `difficulty` 值
| 值 | 说明 |
|----|------|
| `easy` | 简单 |
| `medium` | 中等（默认） |
| `hard` | 困难 |

### `phase` 值
| 值 | 说明 |
|----|------|
| `user_turn` | 等待用户操作 |
| `awaiting_answer` | 等待用户回答系统问题 |
| `awaiting_judgement` | 等待用户判断系统猜测 |
| `finished_win` | 游戏胜利 |
| `finished_loss` | 游戏失败 |

### `answer` 字段值
| 值 | 说明 |
|---|----|
| `yes` | 系统判定为"是" |
| `no` | 系统判定为"否" |
| `unknown` | 系统不确定 |
| `null` | 暂无判定 |

## 错误代码速查

| 代码 | 场景 | 错误消息示例 |
|-----|------|------------|
| 400 | 参数错误 | `"user_word should not be empty"` |
| 400 | 业务违规 | `"Game not found: invalid_game_id"` |
| 400 | 回合状态不符 | `"Invalid turn type for current phase"` |
| 500 | 服务器错误 | `"Failed to create game"` |
| 500 | LLM 调用失败 | `"Failed to submit turn"` |

## 代码片段速查

import requests

game = requests.post(
    "http://localhost:8000/api/v1/games",
    json={"owner_id": "user1", "user_word": "算法", "difficulty": "medium"}

## 常见实现模式

### 模式 1: 完整游戏流程
```
1. POST /games → 获取 game_id
2. LOOP (最多 8 次):
   a. POST /games/{game_id}/turns (提问)
   b. POST /games/{game_id}/turns (回答)
3. 用户猜测 → POST /games/{game_id}/turns
4. 系统反向猜测 (如果前一步错误)
5. 用户判断 → POST /games/{game_id}/turns
6. 根据 phase 判断游戏结束
```

### 模式 2: 错误处理
```python
try:
    response = requests.post(url, json=data, timeout=30)
    if response.status_code >= 400:
        error = response.json()
        print(f"错误: {error['detail']}")
except requests.Timeout:
    print("请求超时，请稍后重试")
except Exception as e:
    print(f"网络错误: {e}")
```

### 模式 3: 轮次锁校验（前端参考）
```
当前 phase = "user_turn"
  → 期望用户提问或猜测 (turn_type="input")
  
当前 phase = "awaiting_answer"
  → 期望用户回答 (turn_type="answer")
  
当前 phase = "awaiting_judgement"
  → 期望用户判断 (turn_type="judge")

如果不符则后端返回错误
```

## 网络配置建议

| 参数 | 建议值 | 说明 |
|-----|------|------|
| 连接超时 | 10s | TCP 连接建立时间 |
| 读超时 | 30-60s | 等待响应时间（可能包含 LLM） |
| 最大重试 | 2-3 次 | 暂时性失败重试 |
| 集中请求限制 | 10/s | 防止过载 |
