❌ 问题 3：GUESS 处理逻辑错误（混淆 user/system intent）

| #   | 问题               | 修复方式                 | 影响            |
| --- | ------------------ | ------------------------ | --------------- |
| 1   | system_action 混乱 | 改用 system_next_action  | 事件格式变更 ⚠️ |

## 三、事件格式变更速记

### 关键变更

```
❌ 不使用：event["system_action"]
✅ 改用：event["system_next_action"]

❌ 返回：{"system_action": "string"}
✅ 改为：{"system_action": "enum", "message": "string"}
```

### 构建事件示例

```python
# 简单意图
event = {"user_intent": "question"}

# 带结果的意图
event = {
    "user_intent": "guess",
    "system_judge": "correct"  # 或 "incorrect"
}

# 带决策的意图
event = {
    "user_intent": "yield_turn",
    "system_next_action": "ask_question"  # 或 "make_guess"
}
```

## 四、状态转换速记表

### 核心转换路径

```
START
  ↓
USER_TURN
  ├─ question  → WAITING_ANSWER
  ├─ guess ✓   → FINISHED
  ├─ guess ✗   → USER_TURN
  ├─ yield_turn(ask)   → WAITING_ANSWER
  ├─ yield_turn(guess) → AWAITING_JUDGEMENT
  └─ invalid   → USER_TURN（轮次锁）

WAITING_ANSWER
  ├─ answer    → USER_TURN
  └─ 其他      → WAITING_ANSWER（轮次锁）

AWAITING_JUDGEMENT
  ├─ judge ✓   → FINISHED
  ├─ judge ✗   → USER_TURN
  └─ 其他      → AWAITING_JUDGEMENT（轮次锁）

FINISHED → FINISHED（结束）
```

## 七、常见错误及修复

### ❌ 错误 1：使用旧的 system_action

```python
# 错误
event = {
    "user_intent": "question",
    "system_action": "ask_question"  # ❌ 错误字段
}

# 正确
event = {"user_intent": "question"}  # ✅ 简单情况不需要 system_action

# 需要系统决策的情况
event = {
    "user_intent": "yield_turn",
    "system_next_action": "ask_question"  # ✅ 改用此字段
}
```

### ❌ 错误 2：检查旧的返回值字段

```python
# 错误
if result["system_action"] == "system_answer":  # ❌ 值是字符串
    pass

# 正确
if result["system_action"] == "answer":  # ✅ 统一的枚举值
    pass
```

### ❌ 错误 3：不检查 is_error

```python
# 危险（忽略错误）
result = fsm.handle_event(bad_event)
state = result["user_next_state"]  # 可能是 ERROR 状态

# 安全（先检查错误）
result = fsm.handle_event(event)
if not result["is_error"]:
    state = result["user_next_state"]
```

### ❌ 错误 4：外部管理轮次

```python
# 不必要的复杂（原版方式）
round_count = 0
for event in events:
    result = fsm.handle_event(event)
    if result["user_next_state"] != "user_turn":
        round_count += 1

# 简单（修复版方式）
for event in events:
    result = fsm.handle_event(event)
    round = fsm.get_round_count()  # 直接查询
```

## 十一、紧急问题排查

### Q: 游戏中断，出现 ERROR 状态？

```python
if result["is_error"]:
    print(result["message"])  # 查看具体错误
    # 检查事件格式是否正确
    print(event)
```

### Q: 轮次没有增加？

```python
# 轮次在用户离开 USER_TURN 时增加
# 检查状态转换：USER_TURN → WAITING_ANSWER 或其他非 USER_TURN
result = fsm.handle_event(event)
print(fsm.get_round_count())  # 应该增加
```

### Q: 一直卡在 WAITING_ANSWER？

```python
# 检查是否发送了 ANSWER 意图
current_state = fsm.get_state()
if current_state == "waiting_answer":
    event = {"user_intent": "answer"}  # 必须是 answer
    result = fsm.handle_event(event)
```

### Q: 游戏没有在 20 轮结束？

```python
# 检查轮次是否被正确计数
remaining = fsm.get_remaining_rounds()
print(f"剩余轮次: {remaining}")

# 如果有问题，检查状态转换是否正确
# 轮次只在 USER_TURN 完成操作时增加
```

## 十三、向后兼容性说明

**修复版不向后兼容原版**，但可以通过中间适配层实现：

```python
class FSMAdapter:
    """适配层，支持原版调用方"""
    def __init__(self):
        self.fsm = GuessGameFSM()  # 使用修复版

    def handle_event(self, event):
        # 转换原版事件格式到新版
        new_event = self._adapt_event(event)
        result = self.fsm.handle_event(new_event)
        # 转换新版返回值到原版
        return self._adapt_result(result)
```
