# 游戏结束判定 - 问题诊断和快速导航

## 🔍 问题诊断总结

**症状:**
```
你的猜测: "我猜是艺术史"
实际响应: ❌ status: active, phase: awaiting_answer
期望响应: ✅ status: finished, result: user_win
```

**根本原因 (按优先级排序):**

| 序号 | 原因 | 位置 | 严重性 |
|------|------|------|------|
| 1 | Intent 识别失败，"猜测"被识别为"回答" | GuessAgent._infer_local_intent() | 🔴 最严重 |
| 2 | 游戏规则要求 awaiting_answer 只允许"回答"，不允许"猜测" | GameService._is_turn_allowed() | 🔴 最严重 |
| 3 | submit_turn 的条件顺序问题，没有优先处理"猜测" | GameService.submit_turn() | 🟠 严重 |
| 4 | _handle_user_guess() 逻辑不完整，没有结束游戏的处理 | GameService._handle_user_guess() | 🟠 严重 |
| 5 | 系统猜测判定没有正确的游戏结束逻辑 | GameService._handle_agent_guess_judgement() | 🟡 中等 |

```python
@staticmethod
def _is_turn_allowed(phase: str, intent: str) -> bool:
    # ✅ 任何非终止阶段都允许猜测
    if intent == "guess":
        return phase != "finished"
    
    # 其他允许的操作
    if phase == "awaiting_answer":
        return intent in {"answer", "guess"}
    if phase == "awaiting_judgement":
        return intent in {"judge", "answer", "guess"}
    if phase == "user_turn":
        return intent in {"question", "guess"}
    return False
```

## ⚠️ 常见坑点

| 坑点 | 症状 | 解决方案 |
|------|------|--------|
| Intent 识别失败 | 猜测被识别为"回答" | 检查 `_extract_guess_word()` 是否正确提取 |
| 游戏规则太严格 | awaiting_answer 拒绝猜测 | 检查 `_is_turn_allowed()` 是否允许当前阶段 |
| 条件顺序不对 | guess_marker 都验证成功但没进结束逻辑 | 检查 submit_turn() 中的条件重排 |
| history 不完整 | 无法追踪游戏历史 | 检查每个 handler 是否都记录了 history item |
| 异常处理缺失 | 解密失败导致崩溃 | 检查 try-except 是否包装了 reveal_target_word() |

## 📞 问题排查流程

遇到问题时，按这个顺序排查：

1. **检查日志** - 看 intent 识别结果
   ```
   INFO: LLM 原始返回 [parse_user_intent]:
   {"intent": "guess" or "answer" or "invalid"}
   ```

2. **检查游戏状态** - 看当前 phase 是什么
   ```
   GET /api/v1/games/{game_id}
   {"phase": "awaiting_answer" or "user_turn" or ...}
   ```

3. **检查条件判断** - 在 submit_turn() 中加断点
   ```
   _is_turn_allowed(phase="awaiting_answer", intent="guess")
   should return True (不是 False)
   ```

4. **检查处理器调用** - 看 _handle_user_guess() 是否被调用
   ```
   logger.info("开始处理用户猜测...")
   ```

5. **检查结束逻辑** - 看 finish_game() 是否被调用
   ```
   INFO: [game_service] 游戏结束: game_id={}, reason=user_win
   ```
