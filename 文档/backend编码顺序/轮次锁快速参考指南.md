# 轮次锁快速参考指南 (Cheat Sheet)

## 快速理解

**轮次锁是什么？**

- 游戏状态机，强制执行用户和系统的交替行动
- 在 `metadata` 中记录当前 `phase`、`next_actor`、`expected_turn_type`
- 在 `submit_turn()` 中进行校验，拒绝违反规则的输入

---

## 三个阶段流转

| 阶段                 | 下一行动者 | 允许意图        | 转移条件                       |
| -------------------- | ---------- | --------------- | ------------------------------ |
| `user_turn`          | 用户       | question, guess | 初始 / 用户回答后 / 用户判断后 |
| `awaiting_answer`    | 用户       | answer          | 系统提问后                     |
| `awaiting_judgement` | 用户       | judge, answer   | 系统猜测后                     |

---

## 校验流程 (submit_turn 中)

```
game.metadata.phase = ?
    ↓
[读取 phase]
    ↓
user_input → parse_user_intent() → intent
    ↓
[校验] _is_turn_allowed(phase, intent) ?
    ├─ Yes → 路由到 handler (answer_question, handle_user_guess 等)
    └─ No  → _reject_turn() → 返回拒绝消息
```

---

## 关键代码片段

### 读取轮次锁状态

```python
turn_lock = service._get_turn_lock_state(game)
# 返回: {"phase": "...", "next_actor": "...", "expected_turn_type": "..."}
```

### 校验是否允许

```python
allowed = service._is_turn_allowed("user_turn", "answer")
# 返回: False (user_turn 不允许 answer)
```

### 状态转换

```python
# 用户回答后返回 user_turn
await repository.update_game_state(
    game_id,
    phase="user_turn",
    next_actor="user",
    expected_turn_type="question",
)
```

## 常见操作矩阵

### 用户在 user_turn 阶段能做什么？

| 用户输入             | 意图     | 结果                          |
| -------------------- | -------- | ----------------------------- |
| "这是一个数学题吗？" | question | ✅ 被处理 (answer_question)   |
| "我猜是 ......"      | guess    | ✅ 被处理 (handle_user_guess) |
| "是"                 | answer   | ❌ 被拒绝                     |
| "正确"               | judge    | ❌ 被拒绝                     |
| "给我提示"           | invalid  | ❌ 被拒绝                     |

### 用户在 awaiting_answer 阶段能做什么？

| 用户输入       | 意图     | 结果                           |
| -------------- | -------- | ------------------------------ |
| "是"           | answer   | ✅ 被处理 (handle_user_answer) |
| "不是"         | answer   | ✅ 被处理 (handle_user_answer) |
| "这是什么题？" | question | ❌ 被拒绝                      |

### 用户在 awaiting_judgement 阶段能做什么？

| 用户输入     | 意图     | 结果                                     |
| ------------ | -------- | ---------------------------------------- |
| "是"         | judge    | ✅ 被处理或游戏结束                      |
| "不是"       | judge    | ✅ 被处理 (handle_agent_guess_judgement) |
| "这是什么？" | question | ❌ 被拒绝                                |

## 修改轮次锁的场景

### 场景 1：需要防止特定意图

1. 找到 `_is_turn_allowed()`
2. 为对应 phase 添加规则

```python
def _is_turn_allowed(self, phase: str, intent: str) -> bool:
    if phase == "user_turn":
        return intent in {"question", "guess"}  # ← 这里修改
    # ...
```

### 场景 2：需要添加新的阶段

1. 在 `_build_initial_metadata()` 中设置初始值
2. 在 `_is_turn_allowed()` 中添加新阶段的规则
3. 在对应的 handler 中设置状态转移

例如，添加 "awaiting_confirmation" 阶段：

```python
# 1. 初始化
def _build_initial_metadata():
    return {
        "phase": "user_turn",
        # ...
    }

# 2. 校验规则
def _is_turn_allowed(self, phase, intent):
    if phase == "awaiting_confirmation":
        return intent in {"judge", "answer"}
    # ...

# 3. 状态转移
# 在某个 handler 中
await self.repository.update_game_state(
    game_id,
    phase="awaiting_confirmation",
    next_actor="user",
    expected_turn_type="judge",
)
```

## 调试技巧

### 验证游戏是否卡在某个阶段

```python
game = await repository.get_game(game_id)
print(game.get("metadata", {}).get("phase"))  # 查看当前阶段
print(game.get("metadata", {}).get("next_actor"))  # 查看应该是谁行动
```

### 模拟用户输入来测试轮次锁

```python
result = await service.submit_turn(
    game_id="test",

    mode="agent",
    turn_type="input"
)
print(result)  # 查看是被处理还是被拒绝
```

```python
if "message" in result and "不是" in result.get("message", ""):
    print("轮次锁生效：", result["message"])
```

## 常见错误及解决

| 错误               | 原因                                   | 解决                                        |
| ------------------ | -------------------------------------- | ------------------------------------------- |
| 用户意图被直接拒绝 | phase 与 intent 不匹配                 | 检查 phase 和期望的 intent 是否对齐         |
| 游戏卡住           | 返回状态没有设置 phase                 | 确保所有 update_game_state 调用都包含 phase |
| 循环提问           | handler 中调用了 decide_agent_action() | 移除自动续问逻辑，让用户主导                |

## API 集成示例

### 前端获取当前阶段

```javascript
// POST /games/{game_id}/turns
// Response:
{
  "game_id": "xxx",
  "status": "active",
  "phase": "user_turn",  // ← 轮次锁信息
  "result": "question_answered",
  "message": "回答完毕，现在轮到你继续提问或直接猜答案。",
  "history": [...]
}

// 根据 phase 决定下一步
if (response.phase === "user_turn") {
  showMessage("轮到你了，请提问或猜测");
} else if (response.phase === "awaiting_answer") {
  showMessage("请回答系统的问题");
}
```
