# 17 接口编写与测试手册

## 1. 文档目标

这份手册的目标是指导你把猜词游戏的**接口层**真正写完整，并且用可执行的方式把它测通。

在前面的手册里，你已经完成了：

- 游戏创建的基础链路
- 猜词核心状态机的设计和部分实现
- prompt 文件职责的拆分
- 仓储层的基础数据访问

这一章要做的是把这些能力真正暴露成稳定接口，并且通过测试验证它们是可用的。

一句话总结：

**这一章不是继续讲概念，而是把“怎么接入、怎么调用、怎么验证”写清楚。**

## 3. 接口编写目标

这一章建议把接口拆成三类来写。

### 3.1 开局接口

开局接口负责：

- 校验用户输入
- 创建新游戏
- 调用小助手生成目标词
- 写入数据库
- 返回开局结果

对应路由：

- `POST /api/v1/games`

### 3.2 回合接口

回合接口负责：

- 接收用户提问
- 接收用户猜测
- 接收用户对系统猜测的确认
- 调用 `GameService.submit_turn()`
- 返回本轮结果和最新状态

对应路由：

- `POST /api/v1/games/{game_id}/turns`

### 3.3 测试接口

测试阶段建议验证：

- 成功创建游戏
- 用户提问可以正常进入回合
- 用户猜测可以触发正确/错误分支
- 系统猜测可以进入等待确认阶段
- 结束状态后不能再继续提交

## 4. 推荐的请求模型

这一部分建议你先写清楚，再写路由。

### 4.1 创建游戏请求模型

当前创建游戏接口可以先保持简单：

```python
from pydantic import BaseModel, Field


class CreateGameRequest(BaseModel):
    owner_id: str | None = Field(default=None, description="游戏创建者ID")
    user_word: str = Field(..., description="玩家输入的开局词")
    difficulty: str | None = Field(default="medium", description="游戏难度")
```

### 4.2 回合请求模型

如果你要让接口真正支持完整猜词流程，建议不要只保留一个 `question` 字段，而是做一个更通用的请求模型。

#### 推荐方案：统一回合模型

```python
from pydantic import BaseModel, Field
from typing import Literal


class SubmitTurnRequest(BaseModel):
    turn_type: Literal["input", "judge"] = Field(default="input", description="回合类型")
    content: str | None = Field(default=None, description="用户输入内容")
    judgement: bool | None = Field(default=None, description="用户是否确认系统猜测正确")
    mode: str = Field(default="agent", description="回答模式")
```

#### 字段说明

- `turn_type = input`：表示用户主动输入问题或猜测
- `turn_type = judge`：表示用户在判断系统猜测是否正确
- `content`：普通输入内容
- `judgement`：系统猜测的对错判断
- `mode`：保留给主模型 / 小助手切换使用

### 4.3 如果你想先保持最小改动

也可以继续沿用现在的 `question` 字段，但要在文档里明确：

- 这个字段其实承载了“提问、猜测、确认”三种语义
- 真正的分类由 `GuessAgent.parse_user_intent()` 和 `GameService` 进行

但从长期维护角度看，统一回合模型更稳。

## 5. 推荐的响应模型

接口返回也建议分成“开局响应”和“回合响应”两类。

### 5.1 开局响应模型

```python
from datetime import datetime
from pydantic import BaseModel, Field


class CreateGameResponse(BaseModel):
    game_id: str = Field(..., description="游戏ID")
    status: str = Field(..., description="游戏状态")
    round_count: int = Field(..., description="当前轮次")
    user_word: str = Field(..., description="玩家输入的开局词")
    difficulty: str = Field(..., description="游戏难度")
    system_word_encrypted: str = Field(..., description="系统词密文")
    created_at: datetime = Field(..., description="创建时间")
```

### 5.2 回合响应模型

建议回合接口返回这些字段：

- `game_id`
- `status`
- `phase`
- `result`
- `message`
- `history`
- `question`
- `agent_guess`

#### 示例结构

```python
from pydantic import BaseModel, Field


class SubmitTurnResponse(BaseModel):
    game_id: str = Field(..., description="游戏ID")
    status: str = Field(..., description="当前状态")
    phase: str = Field(..., description="当前阶段")
    result: str = Field(..., description="当前结果")
    message: str = Field(..., description="提示信息")
    history: list[dict] = Field(default_factory=list, description="历史记录")
```

### 5.3 为什么建议把 phase 返回给前端

因为前端需要知道当前该展示什么：

- `user_turn`：让用户继续提问或猜测
- `agent_turn`：展示系统刚刚提问的结果
- `awaiting_judgement`：让用户判断系统猜测对不对
- `finished`：显示游戏结束

## 6. 路由层编写建议

### 6.1 开局接口路由

建议继续保留在 `games.py` 里。

#### 推荐结构

```python
@router.post("", response_model=CreateGameResponse, status_code=status.HTTP_201_CREATED)
async def create_game(request: CreateGameRequest, service: GameService = Depends(get_game_service)):
    try:
        return await service.create_game(
            owner_id=request.owner_id,
            user_word=request.user_word,
            difficulty=request.difficulty,
        )
    except ValueError as exc:
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(exc))
    except Exception:
        raise HTTPException(status_code=status.HTTP_500_INTERNAL_SERVER_ERROR, detail="Failed to create game")
```

### 6.2 回合接口路由

建议在 `submit_turn.py` 中把路由逻辑写成“只负责转发”。

#### 推荐结构

```python
@router.post("/{game_id}/turns")
async def submit_turn(game_id: str, request: SubmitTurnRequest, service: GameService = Depends(get_game_service)):
    try:
        return await service.submit_turn(game_id, request.question, request.mode)
    except ValueError as exc:
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(exc))
    except Exception:
        raise HTTPException(status_code=status.HTTP_500_INTERNAL_SERVER_ERROR, detail="Failed to submit turn")
```

### 6.3 路由层的边界

路由层只做三件事：

- 接收参数
- 调用 service
- 统一转 HTTP 异常

不要把这些东西写进路由层：

- prompt 拼接
- 业务状态切换
- 数据库更新
- 猜中判断

## 7. Service 层和接口的对应关系

接口是否好用，最终取决于 `GameService` 能不能稳定给出结果。

### 7.1 `create_game()` 对应开局接口

作用：

- 调用小助手生成目标词
- 生成加密材料
- 创建数据库记录
- 返回开局信息

### 7.2 `submit_turn()` 对应回合接口

作用：

- 读取游戏状态
- 判断当前 `phase`
- 处理用户提问、用户猜测、系统猜测确认
- 写入历史和状态
- 返回下一步提示

### 7.3 `end_game()` 对应结束场景

作用：

- 标记游戏结束
- 写入结束原因
- 写入结束时间

这类方法通常可以保留给手动结算或异常收口使用。

### 8.1 开局接口完整骨架

```python
from datetime import datetime
from pathlib import Path

from fastapi import APIRouter, Depends, HTTPException, status
from pydantic import BaseModel, Field

from app.agents.guess_agent import GuessAgent
from app.agents.utils.memory_policy import MemoryPolicy
from app.agents.utils.prompt_loader import PromptLoader
from app.core.config import get_settings
from app.core.database import get_database
from app.repositories.game_repository import GameRepository
from app.services.game_service import GameService


class CreateGameRequest(BaseModel):
    owner_id: str | None = Field(default=None)
    user_word: str = Field(...)
    difficulty: str | None = Field(default="medium")


class CreateGameResponse(BaseModel):
    game_id: str
    status: str
    round_count: int
    user_word: str
    difficulty: str
    system_word_encrypted: str
    created_at: datetime


router = APIRouter(prefix="/games", tags=["games"])


def get_game_service() -> GameService:
    settings = get_settings()
    repository = GameRepository(get_database())
    prompt_loader = PromptLoader(str(Path(__file__).resolve().parents[2] / "agents" / "prompt"))
    memory_policy = MemoryPolicy(max_rounds=8)
    agent = GuessAgent(settings, prompt_loader, memory_policy)
    return GameService(settings, agent=agent, repository=repository, memory_policy=memory_policy)


@router.post("", response_model=CreateGameResponse, status_code=status.HTTP_201_CREATED)
async def create_game(request: CreateGameRequest, service: GameService = Depends(get_game_service)):
    try:
        return await service.create_game(
            owner_id=request.owner_id,
            user_word=request.user_word,
            difficulty=request.difficulty,
        )
    except ValueError as exc:
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(exc))
    except Exception:
        raise HTTPException(status_code=status.HTTP_500_INTERNAL_SERVER_ERROR, detail="Failed to create game")
```

## 9. 测试设计

这一部分是最关键的，因为你现在不只是要写接口，还要确认它真的能工作。

### 9.1 测试分层

建议把测试分成三层：

- 手工接口测试
- Service 层逻辑测试
- Repository 层数据测试

### 9.2 推荐测试顺序

先测最外层，再测最内层：

1. `/api/v1/games` 是否能创建游戏
2. `/api/v1/games/{game_id}/turns` 是否能接收输入
3. `GameService.submit_turn()` 是否能正确切状态
4. `GameRepository.update_game_state()` 是否能写回数据
5. `GuessAgent` 的 JSON 输出是否能被解析

## 15. 你现在最该先做什么

1. 先把 `SubmitTurnRequest` 和回合返回结构定下来
2. 再确认 `submit_turn.py` 和 `GameService.submit_turn()` 的参数一致

## 16. 一句话总结
