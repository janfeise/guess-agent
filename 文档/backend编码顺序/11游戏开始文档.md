# 11 游戏开始文档

## 1. 文档目标

这一章用于定义“游戏开始”这一步应该怎么做，以及后端应该如何在开局时同时完成用户词和系统词的加密。

本阶段的核心要求是：

- 用户输入一个 word，记为 `user_word`
- 系统从词库中随机选择一个 word，记为 `system_word`
- `user_word` 不需要加密，直接保存为明文或规范化文本即可
- `system_word` 需要加密后再保存
- 接口必须禁止返回 `system_word` 的明文
- 接口禁止在开局时返回随机加密密码
- 系统词对应的随机加密密码只能在游戏结束后返回

这一章只做“游戏开始”，不展开后续回合问答。

---

## 2. 游戏开始时要做什么

游戏开始时，后端要完成下面几件事：

1. 校验用户输入的 `user_word`
2. 从 `constants.py` 的词库里随机选择一个 `system_word`
3. 生成一段随机密码，用于系统词加密
4. 保存用户词明文或规范化文本
5. 对 `system_word` 加密
6. 把加密后的结果写入数据库
7. 返回游戏开始结果，但不返回系统密码明文

### 2.1 为什么要这样做

因为游戏一开始就要保证两个原则：

- 明文不能直接落库
- 系统选中的词不能直接返回给前端

这样才能保证游戏公平性和后续验证的可控性。

## 4. 游戏开始的数据约定

### 4.1 用户输入

建议用户开局时至少提交：

- `user_word`：用户输入的 word

如果后面需要更强的验证能力，也可以继续扩展：

- `owner_id`
- `user_password`

### 4.2 系统生成

系统开局时要自动生成：

- `system_word`：从词库随机选出的词
- `system_password`：随机生成的加密密码

### 4.3 返回给前端

接口返回时建议至少包含：

- `game_id`
- `user_word`
- `system_word_encrypted`
- `status`
- `created_at`

注意：

- 不返回 `system_word` 明文
- 不返回 `user_word` 以外的任何用户敏感信息
- 不返回 `system_password`
- 如果需要保留系统密码，应以加密后的形式存入数据库，等游戏结束后再返回

## 5. 推荐实现流程

### 5.1 校验用户词

先调用 `validate_word(user_word)`，确认：

- 不能为空
- 长度不超过 10 字

如果后面你要把词池从水果词改成知识词，校验方式不变，只需要替换词库内容。

### 5.2 随机选择系统词

从 `SYSTEM_WORDS` 中随机取一个词，作为 `system_word`。

### 5.3 生成随机密码

为系统词生成一个随机密码，例如：

- `secrets.token_urlsafe(16)`

这个密码用于加密系统词，但不能在开局接口里返回给调用方。
建议将这个密码再用服务端密钥加密后保存到数据库，等游戏结束后再解密返回。

### 5.4 保存用户词并加密系统词

分别执行：

- `encrypt(system_word, system_password)`

用户词不需要加密，但建议先做规范化，再直接保存为明文或标准化文本。

### 5.5 写入 MongoDB

把加密后的结果写入 `games` 集合，保存为一局完整游戏记录。

### 5.6 返回结果

返回时只给前端必要信息，不返回任何系统词明文，也不返回 `system_password`。

## 7. 详细代码实现

下面给出建议实现骨架。

### 7.1 constants.py 的推荐补充

当前 `constants.py` 已有词库和校验函数，如果你要让“游戏开始”更清晰，可以补一个系统词选择函数。

```python
# backend/app/core/constants.py

import random

MAX_WORD_LENGTH = 10

SYSTEM_WORDS = [
    "苹果",
    "香蕉",
    "橘子",
    "葡萄",
    "西瓜",
    "草莓",
    "梨子",
    "桃子",
    "柠檬",
    "菠萝",
    "芒果",
    "蓝莓",
]


def normalize_word(word: str) -> str:
    return word.strip().lower()


def validate_word(word: str) -> str:
    normalized_word = normalize_word(word)
    if len(normalized_word) == 0:
        raise ValueError("词不能为空")
    if len(normalized_word) > MAX_WORD_LENGTH:
        raise ValueError(f"输入的词语长度不能超过 {MAX_WORD_LENGTH} 个字符")
    return normalized_word


def pick_system_word() -> str:
    if not SYSTEM_WORDS:
        raise ValueError("系统词库为空")
    return random.choice(SYSTEM_WORDS)
```

#### 设计说明

- `validate_word()` 负责用户输入校验
- `pick_system_word()` 负责系统随机选词
- 两者职责分离，后续更容易维护

### 7.2 game_service.py 的推荐实现

建议在 `GameService` 里新增一个“开始游戏”的方法，专门处理开局逻辑。

```python
# backend/app/services/game_service.py

import secrets
import uuid
from datetime import datetime

from app.agents.utils.streaming import build_stream_chunk, build_stream_end
from app.core.constants import pick_system_word, validate_word
from app.core.security import encrypt


class GameService:
    def __init__(self, settings, agent, repository, memory_policy):
        self.settings = settings
        self.agent = agent
        self.repository = repository
        self.memory_policy = memory_policy

    async def create_game(self, owner_id=None, user_word: str | None = None):
        if not user_word:
            raise ValueError("user_word is required")

        normalized_user_word = validate_word(user_word)
        system_word = pick_system_word()
        system_password = secrets.token_urlsafe(16)
        now = datetime.now()
        game_id = str(uuid.uuid4())

        system_word_encrypted = encrypt(system_word, system_password)
        system_password_encrypted = encrypt(system_password, self.settings.encryption_secret)

        game_doc = {
            "game_id": game_id,
            "owner_id": owner_id,
            "status": "active",
            "history": [],
            "summary": "",
            "round_count": 0,
            "created_at": now,
            "updated_at": now,
            "metadata": {},
            "user_word": normalized_user_word,
            "system_word_encrypted": system_word_encrypted,
            "system_password_encrypted": system_password_encrypted,
            "encryption_metadata": {
                "algorithm": "fernet",
                "created_at": now,
                "password_source": "secrets.token_urlsafe",
            },
        }

        await self.repository.create_game(game_doc)

        return {
            "game_id": game_id,
            "status": "active",
            "round_count": 0,
            "created_at": now,
            "user_word": normalized_user_word,
            "system_word_encrypted": system_word_encrypted,
        }
```

#### 设计说明

- `user_word` 先校验再保存
- `system_word` 从词库随机选出
- `system_password` 用随机字符串生成
- `system_password_encrypted` 保留给后续验证
- 返回结果只包含用户词明文和系统词密文，不包含 `system_password`

#### 为什么不开局返回 `system_password`

因为系统词是用这个随机密码加密的，但游戏开始时不能直接暴露这个密码。

如果后续要做验证或解密，建议把这个密码以加密形式保存在数据库中，等游戏结束后再解密返回。

#### 为什么用户词不加密

因为用户词本身是用户主动输入的开局内容，不属于需要隐藏的谜底。

当前阶段更重要的是：

- 保证系统词不泄露
- 保证系统密码不在开局返回
- 保证后续能够在结束时解密校验

#### 为什么不返回系统词明文

因为这是游戏规则要求：系统选择的词不能直接暴露给接口调用方。

### 7.3 game_repository.py 的推荐实现

Repository 只负责把开局记录写入数据库。

```python
# backend/app/repositories/game_repository.py

from datetime import datetime


class GameRepository:
    def __init__(self, db):
        self.db = db
        self.collection = self.db["games"]

    async def create_game(self, game_doc: dict):
        existing_game = await self.collection.find_one({"game_id": game_doc["game_id"]})
        if existing_game:
            raise ValueError(f"Game with id {game_doc['game_id']} already exists")

        await self.collection.insert_one(game_doc)
        return game_doc

    async def get_game(self, game_id: str):
        return await self.collection.find_one({"game_id": game_id})
```

#### 设计说明

- 开局文档一次插入
- 任何重复 `game_id` 都直接拒绝
- Repository 不处理加密，只处理落库
- `user_word` 以明文或规范化文本保存
- `system_word_encrypted` 和 `system_password_encrypted` 以密文保存

### 7.4 games.py 的推荐实现

接口层负责接收请求并调用 `GameService.create_game()`。

```python
# backend/app/api/v1/games.py

from datetime import datetime

from fastapi import APIRouter, Depends, HTTPException, status
from pydantic import BaseModel, Field

from app.agents.utils.memory_policy import MemoryPolicy
from app.core.config import get_settings
from app.core.database import get_database
from app.repositories.game_repository import GameRepository
from app.services.game_service import GameService


class CreateGameRequest(BaseModel):
    owner_id: str | None = Field(default=None, description="游戏创建者的用户ID，可以为空")
    user_word: str = Field(..., description="用户输入的词，长度不超过10个字符")


class CreateGameResponse(BaseModel):
    game_id: str = Field(..., description="游戏ID")
    status: str = Field(..., description="游戏状态")
    round_count: int = Field(..., description="当前回合数")
    created_at: datetime = Field(..., description="游戏创建时间")
    user_word: str = Field(..., description="用户输入的词，原样返回或返回规范化文本")
    system_word_encrypted: str = Field(..., description="系统词加密后的密文")


router = APIRouter(prefix="/games", tags=["games"])


def get_game_service() -> GameService:
    settings = get_settings()
    repository = GameRepository(get_database())
    memory_policy = MemoryPolicy(max_rounds=8)
    return GameService(settings, agent=None, repository=repository, memory_policy=memory_policy)


@router.post("", response_model=CreateGameResponse, status_code=status.HTTP_201_CREATED)
async def create_game(request: CreateGameRequest, service: GameService = Depends(get_game_service)):
    try:
        return await service.create_game(owner_id=request.owner_id, user_word=request.user_word)
    except ValueError as exc:
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(exc))
    except Exception:
        raise HTTPException(status_code=status.HTTP_500_INTERNAL_SERVER_ERROR, detail="Failed to start game")
```

#### 设计说明

- 请求里显式传 `user_word`
- 响应里只返回加密结果，不返回系统密码
- 不返回系统词明文
- 路由只做转发，不做业务处理

## 11. 本章结论

这一章的目标很简单：

- 开局时保存用户词明文或规范化文本
- 开局时随机选择并加密系统词
- 开局时不返回系统词对应的随机密码
- 只有游戏结束后才返回系统词对应的随机密码
- 绝不返回系统词明文

只要这一步做好，后面的回合问答和验证流程就有了稳定的基础。
