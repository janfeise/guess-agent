# 后端结构设计说明（重点理解）

## 分层思想

```text
API层 → Service层 → Agent层 → Repo层 → DB
```

### API层

- 只负责接收请求 & 返回响应
- 不写业务逻辑

------

### Service层（核心）

- 游戏流程控制
- 调用 Agent / 加密 / 数据库

例如：

```python
def ask_question(game_id, question):
    # 1. 获取游戏状态
    # 2. 调用 Agent 推理
    # 3. 记录结果
    # 4. 返回回答
```

------

### Agent层（最关键）

这里是你的“灵魂部分”

```bash
agents/
├── guess_agent.py   # 控制AI行为
├── prompt/          # Prompt设计
└── tools/           # 可扩展工具
```

👉 后期你所有“智能升级”，都在这里发生

------

### Repository层

- 专门操作 MongoDB
- 避免业务代码直接操作数据库

# 推荐开发顺序（非常重要）

```text
1. backend/main.py 跑起来
2. 写一个 /chat 接口（直连 DeepSeek）
3. 实现 game_service（最小逻辑）
4. 前端搭一个 ChatBox
5. 打通一轮问答
6. 加入加密模块
7. 引入 MongoDB
8. 最后接入 LangChain
```
